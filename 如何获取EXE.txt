  📦 Windows EXE 自动构建说明
================================================================

✅ 代码已推送到 GitHub，正在自动构建两个 exe 文件！

================================================================
  下载构建好的 EXE (约 5-10 分钟后)
================================================================

⚠️ 重要提示：你的代码在 GitLab 上，需要使用 GitLab CI/CD

方式一：GitLab CI/CD (需要 Windows Runner)
-------------------------------------------
1. 访问你的 GitLab 仓库
   https://gitlab.stardustgod.com/wanyihao/rs_ios

2. 点击左侧菜单的 "CI/CD" > "Pipelines"

3. 找到最新的 Pipeline
   - ⚠️ 如果显示 "This job is stuck"，说明没有可用的 Windows Runner
   - ✅ 如果构建成功，会显示绿色对勾

4. 点击 Pipeline，进入 Job 详情

5. 点击右侧的 "Browse" 按钮查看产物

6. 下载 exe 文件:
   - dist/Android性能监控.exe
   - dist/跨平台性能监控.exe

方式二：本地手动打包 (如果 GitLab 没有 Windows Runner)
-------------------------------------------
需要在 Windows 电脑上执行:

1. 安装依赖:
   pip install -r requirements_windows.txt
   pip install pyinstaller

2. 打包 Android 版:
   pyinstaller build_android.spec

3. 打包统一版:
   pyinstaller build_unified.spec

4. 在 dist\ 目录找到生成的 exe 文件

================================================================
  使用 EXE
================================================================

Android性能监控.exe (推荐):
  - 双击运行
  - 连接 Android 设备 (开启 USB 调试)
  - 自动打开浏览器 http://localhost:5003
  - 选择应用开始监控

跨平台性能监控.exe:
  - 支持 iOS + Android
  - iOS 需要安装 iTunes

================================================================
  系统要求
================================================================

✓ Windows 7/8/10/11 (64位)
✓ Android SDK Platform Tools (ADB)
✓ Android 设备开启 USB 调试

================================================================

构建配置文件:
- .github/workflows/build-windows.yml (GitHub Actions)
- build_android.spec (Android 版配置)
- build_unified.spec (完整版配置)
